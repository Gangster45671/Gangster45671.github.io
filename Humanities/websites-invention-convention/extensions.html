Ajax+Canvas: Extensions

After Javascript was added some more powerfull scripting and better graphing capabilities with canvas were implemented. To the right you can see a HTML and JS example of how the canvas element can be used, it is not editable, but you should be able to understand it. By clicking the button, it will draw a random circle. These graphic elements meant that screens could be drawn in real time.
Ajax is a extension to Javascript and it's fully compatible with it. It adds extra features, for example being able to actively target objects and accesing databases from websites. It stands for Asynchronous JavaScript And XML.

This let websites load user information from databases without having reloading the whole page, which allowed for live updates.
The canvas meant that graphing, and other more advanced graphics could be done in browser.
Ajax made all the websites you know of today, fast and responsive. While in the old days, when you submited a form you had to wait for the entire page to reload. Now it can be done nearly instantly. And your Twitter feed can load new items, by just 'fetching' them. No reload needed.
The new canvas element, upon many things; started another era of web games. With Flash player now being slowly phased out, canvas was the replacement. Try a canvas based game here.
